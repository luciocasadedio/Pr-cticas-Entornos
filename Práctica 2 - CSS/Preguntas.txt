1. CSS son las siglas de hojas de estilo en cascada, es una tecnología que permite controlar la apariencia de una página web programada en HTML.
Se pueden especificar estilos como el tamaño, la fuente, el color, disponer texto, etc. NO ES UN LENGUAJE DE PROGRAMACIÓN.

2. Las reglas de estilo funcionan aplicando una forma de presentación mediante una sintaxis compuesta por un selector que le indica al navegador 
que parte del documento o etiqueta HTML se modificará, los selectores se agrupan separando por comas y abriendo llave para luego poner la propiedad que indica que aspecto visual se modificará
como por ejemplo, color/font-family poniendo dos puntos, y luego el valor que establece el parámetro que se asigna a dicha propiedad.

3. Las 3 formas de darle estilo a un documento son: a nivel de elemento HTML definiendo la propiedad style en su interior, a nivel de página agrupándo los estilos dentro
de una etiqueta <style> en el head, o en un archivo independiente agrupando todas las reglas de estilo en un archivo .css.

4.Los selectores más comunes son los de etiqueta/tipo como p, el selector de clase "." que selecciona uno o varios elementos que compartan el mismo atributo de clase, y 
el selector de identificador # que selecciona un elemento único y específico en la página que posea ID. 

5. Una pseudo-clase es una palabra clave que se añade al final de un selector (precedida por dos puntos :) para especificar un estado especial del elemento seleccionado. 
No selecciona un elemento por lo que es, sino por cómo interactúa el usuario con él o por su posición en la estructura. Ej: a:link, a:visited, a:hover.

6. La herencia en CSS es el comportamiento mediante el cual ciertas propiedades de diseño aplicadas a un elemento "padre" (el contenedor) se transmiten de forma automática a los elementos "hijos" que están dentro de él.

7. La "cascada" es el algoritmo fundamental que los navegadores utilizan para resolver conflictos. 
Cuando existen varias reglas de CSS diferentes que intentan aplicar estilos distintos a un mismo elemento HTML, la cascada determina cuál es la regla que "gana" y, por lo tanto, se dibuja en pantalla.
Este proceso tiene en cuenta 3 factores, 1. la importancia y origen (reglas marcadas con !important tienen más prioridad) 2. Un selector más específico vence a uno menos específico. Por ejemplo,
un estilo aplicado a un #id gana sobre un estilo aplicado a una .clase y la clase gana sobre la etiqueta. (De más específico a más general) 3. Orden de aparición, si dos reglas tienen la misma
especificidad prevalecerá la última regla que el navegador lea (la que este más abajo o en el último archivo css cargado)
