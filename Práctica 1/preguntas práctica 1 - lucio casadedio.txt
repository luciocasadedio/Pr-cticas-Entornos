Ejercitación 1:
1. ¿Qué es HTML, cuándo fue creado, cuáles fueron las distintas versiones y cuál es la última?
HTML es el lenguaje estándar utilizado para publicar en la Web. 
Fue creado a principios de los años 90. 
Pasó por varias versiones como HTML 2.0, HTML 3.2 y la familia HTML 4 (4.0 y 4.01). 
La última versión y el estándar actual es HTML5.

2. ¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos con HTML?
La especificación del W3C marca tres principios fundamentales:

Separar estructura y presentación: Usar HTML solo para la estructura y dejar el diseño visual a las hojas de estilo (CSS).

Considerar la accesibilidad universal: Asegurar que la web pueda ser leída por todos, incluyendo agentes de usuario no visuales.

Ayudar con la representación incremental: Escribir el código de forma que los navegadores puedan ir mostrando la página mientras se descarga, sin tener que esperar a que termine.

3. En las Especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿Y obsoleto?
Un elemento está desaprobado cuando ya no se recomienda su uso, ya sea porque fue reemplazado por un tag más moderno (ej. <object> en vez de <applet>) o porque su función visual debe hacerse con hojas de estilo.
Por otro lado, se considera obsoleto cuando el elemento fue eliminado definitivamente de la especificación y no hay garantías de que el navegador lo pueda interpretar.

4. ¿Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?
El DTD es el conjunto de reglas que le indica al navegador qué etiquetas y estructuras están permitidas en el documento. HTML 4.01 contempla tres DTDs:

DTD Estricto: Es el más limpio, no permite elementos ni atributos desaprobados (obliga a usar hojas de estilo).

DTD Transicional: Permite el uso de elementos y atributos desaprobados, pensado para sitios más antiguos.

DTD con Marcos (Frameset): Es igual al Transicional, pero le suma la posibilidad de dividir la pantalla usando el elemento <frameset>.

5. ¿Qué son los metadatos y cómo se especifican en HTML?
Los metadatos son "información sobre el documento" (por ejemplo, el idioma, el autor, palabras clave). Es información que no se ve directamente en la pantalla pero sirve para los navegadores y buscadores. 
Se especifican en la cabecera del documento (dentro del bloque <head>) usando la etiqueta <meta>.

<p><label><input type="radio" name="opcion" id="X" value="X" />X</label><br /> --> 
<label><input type="radio" name="opcion" id="Y" value="Y" />Y</label></p>
Es un input que al estar bajo el mismo nombre, el navegador los trata como un grupo: si seleccionás "X", automáticamente se desmarca "Y". 

<p><label><input type="radio" name="opcion1" id="X" value="X" />X</label><br />
<label><input type="radio" name="opcion2" id="Y" value="Y" />Y</label></p>
Aquí cada botón tiene un nombre diferente, name="opcion1" y name="opcion2". 
Como no pertenecen al mismo grupo, el navegador permite que ambos queden seleccionados al mismo tiempo, lo cual rompe la lógica habitual de un botón "radio" (que debería ser de opción única)

<select name="lista[]" multiple="multiple">
<optgroup label=" Caso 1">
<option>Mayo</option>
<option>Junio</option>
</optgroup>
<optgroup label=" Caso 2">
<option>Mayo</option>
<option>Junio</option>
</optgroup>
</select>

Es un menú desplegable estándar donde solo se puede elegir una opción a la vez.
Al incluir el atributo multiple="multiple", el navegador permite al usuario seleccionar varias opciones simultáneamente
Además, el nombre de la lista cambia a lista[] (notación de array) para que el servidor pueda recibir todos los valores seleccionados
Ambas utilizan la etiqueta <optgroup> con el atributo label ("Caso 1" y "Caso 2") para crear categorías o títulos dentro del menú que no son seleccionables, 
pero sirven para organizar visualmente las opciones <option>.
